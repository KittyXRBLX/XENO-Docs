Stage 01

Initialization

To begin, load the library and define your basic configuration. This ensures the environment is ready before elements are added.

local XENO = loadstring(game:HttpGet('...'))()

local XENOGUINAME = "Xeno Panel"
wait(6) -- Essential for stability
Stage 02

Theming & Style

XENO comes with built-in color profiles. You can set a global theme or add manual toggle buttons for users to choose their own style.

-- Set a default theme
XENO:SetTheme("Purple") 

-- Add theme selection buttons
XENO:AddThemeBtn("Purple", Themes.Purple.Accent1)
XENO:AddThemeBtn("Green", Themes.Green.Accent1)
XENO:AddThemeBtn("Blue", Themes.Blue.Accent1)
Stage 03

Structure

Organize your UI. Add a search bar for quick navigation and headers to categorize your features.

🔍 AddSearchBar()

XENO:AddSearchBar()

🏷️ AddHeader(title)

XENO:AddHeader("Combat Features")
Stage 04

Basic Controls

Buttons and Switches are the backbone of any script. The third parameter notify determines if a toast alert appears.

🔘 AddButton(label, callback, notify)

XENO:AddButton("Click Me", function() print("Hi") end, true)

🔀 AddSwitch(label, callback, notify)

XENO:AddSwitch("Auto Farm", function(s) print(s) end, true)
Stage 05

Variables & Ranges

Use Sliders for numbers and TextBoxes for custom strings. Both support anti-spam notifications.

🎚️ AddSlider(label, min, max, default, callback, notify)

XENO:AddSlider("WalkSpeed", 16, 100, 16, function(v) ... end, true)

✏️ AddTextBox(label, placeholder, default, callback, notify)

XENO:AddTextBox("Player", "Name...", "", function(t) ... end, false)
Stage 06

Hotkeys

Let users trigger functions using their keyboard. Displayed clearly in the sidebar for easy reference.

⌨️ AddKeybind(label, keyCode, callback, notify)

XENO:AddKeybind(
    "Toggle UI", 
    Enum.KeyCode.RightControl, 
    function() ToggleUI() end, 
    true
)
